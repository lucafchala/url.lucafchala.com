# url.lucafchala.com

> The public, read‑only directory of every PURL (permanent short link) in the `lucafchala.com` network, plus a quick‑launch grid of all the ecosystem's pages.

**Live:** [url.lucafchala.com](https://url.lucafchala.com) · **Stack:** static HTML/CSS/JS on Cloudflare Pages · **Build step:** none

Part of the [lucafchala.com ecosystem](https://github.com/lucafchala/lucafchala.com#the-ecosystem). Design system, shared conventions, and the redirect data flow are documented in the [hub README](https://github.com/lucafchala/lucafchala.com#readme).

---

## What it is

**One sentence:** `url.lucafchala.com` is a static, single‑page, read‑only listing of all short links (PURLs) defined for `lucafchala.com`, rendered client‑side from a `data.json` file, with copy/open buttons and a grid of links to every sibling site.

**In a paragraph:** This site is the human‑facing companion to the machine‑facing `_redirects` file. The actual redirecting (`/<slug>` → destination) is handled by Cloudflare Pages via `_redirects`; this page exists so a visitor can *browse and copy* those links. Both `_redirects` and `data.json` here are **generated and committed by [`dash`](https://github.com/lucafchala/dash.lucafchala.com)** — this repo is an output target, not something you edit by hand. The page fetches `/data.json` at load, groups the links, and renders them; it never writes anything.

---

## Architecture

- **Host:** Cloudflare Pages, static files from the repo root. Push to `main` → auto‑deploy.
- **Rendering:** `index.html` fetches `/data.json`, groups the entries (`contact`, `events`, `video`, `tools`), and builds the DOM. `events` and `video` start collapsed.
- **Resilience:** if the local `/data.json` fetch fails (e.g. a fresh/empty deploy), it falls back to `https://dash.lucafchala.com/data.json`, so the page is never empty as long as the dash repo has data.
- **Redirects:** `_redirects` performs the real 301s at the edge; the page itself only displays them.
- **No backend, no env vars, no secrets.**

---

## How it works

```
visitor → index.html
            ├─ fetch('/data.json')                         (primary)
            │     └─ on failure → fetch dash.lucafchala.com/data.json   (fallback)
            ├─ group by {contact, events, video, tools}
            └─ render rows with [copiar] (copy https://url.lucafchala.com/<slug>) and [abrir] (open destination)
```

- **`data.json`** — the source list, shape `{ "redirects": [ { "slug", "destination", "group" } ] }`.
- **`_redirects`** — Cloudflare Pages rules, `/<slug>  <destination>  301`, that actually redirect visitors who hit `url.lucafchala.com/<slug>`.
- **Hub grid** — the `PÁGINAS` section is a hardcoded set of `.hub` cards linking to `lucafchala.com`, `fotos`, `radio`, `weblog`, `now`, `paste`, `dash`, `keys`, `proof`, `status`. Edit these in `dash`'s `genUrlIndex()`, since `index.html` here is regenerated on the next PURL save.

## Prerequisites

- A text editor and `git` for any manual tweak.
- (Optional) a static server for local preview (`python3 -m http.server`, `npx serve`, Live Server).
- To regenerate content: the [`dash`](https://github.com/lucafchala/dash.lucafchala.com) control panel with a configured GitHub PAT (this repo is *optional* in the dash settings — `gh_repo_url`).

No accounts or environment variables are required to run or deploy this repo itself.

## Install & deploy

```bash
git clone https://github.com/lucafchala/url.lucafchala.com.git
cd url.lucafchala.com
python3 -m http.server 8000      # preview at http://localhost:8000
# … then commit & push to main; Cloudflare Pages deploys automatically
git push origin main
```

> Normally you don't push here directly. Editing a short link in `dash` and saving writes `_redirects`, `data.json`, `index.html`, and `404.html` into this repo for you (sequentially, to avoid GitHub `409`s) and the deploy follows.
